What is in the BeeWare suite of tools?

{% endblock %} {% block body %}

BeeWare isn't a single package or tool. It's an entire suite of tools. Each tool performs exactly one task in the app development process, with the output of one tool forming the input to the next.

Each project of the BeeWare suite fills one of the following 6 roles:

Application

A standalone end-user application. Although applications are written in Python, this is an implementation detail - you shouldn't ever be aware of this fact. Applications are installed and run the same way as any other end-user application on your computer.

Tools

A special class of application that is installed into a Python virtual environment. Tools are used to assist when developing standalone applications, in roles from debugging and testing to packaging.

Libraries

A re-usable block of code that can be used in a project. Libraries will usually be installed as dependencies of tools, applications, or other libraries.

Bridge

A bridge is a combination tool & library that is used to run Python code where the CPython runtime cannot be used. The bridge provides the mechanism to run Python code in a different runtime environment.

Templates

Cookiecutter templates for the boilerplate code needed to get a Python project running as an application on a particular platform.

Support

Pre-packaged bundles of compiled artefacts that are needed to run on a particular platform.
